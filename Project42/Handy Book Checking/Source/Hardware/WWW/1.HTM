...are interesting animals.

Using only 4 pins of the 5-pin connector, plus a built-in microcontroller, you have an intelligent keyboard capable of many things.

The keyboard connector signal definition can be found in any PC reference book. However, here's the pinout for the "standard" keyboard connector, looking INTO THE PLUG: 

     1       3
      4     5
         2


 1 - Keyboard clock
 2 - Data (bi-directional line)
 3 - Reset (active low)
 4 - GND
 5 - +5V

I don't know what the PS/2 connector pinout is, offhand.

As for the keycodes: the keyboard generates "scan codes" which the keyboard microcontroller sends to the computer's motherboard decoder microcontroller. These "scan codes" DO NOT DIRECTLY "translate" into ASCII character codes - the decoder translates those codes into the appropriate ASCII character code, representing the key pressed.

The FUNCTION keys are a special case: they generate TWO scan codes which the decoder translates into the appropriate key. Similar for the SHIFT, CTRL, ALT, editing (i.e Page Up, Insert) and "cursor" (keypad) keys.

I remember the magazine Circuit Cellar INK had an EXCELLENT series of articles on the keyboard a number of years ago. Visit "www.circuitcellar.com" and see if they have that information online.

As for "coding", you can DIRECTLY access the keyboard via PC BIOS routines (INT 16h services). Assuming you're using a PC architecture machine, you can get lots of "programming guides" for dealing with BIOS and DOS services for accessing the keyboard. If you're trying to write a "custom driver" for another application, you need to get the Circuit Cellar INK back issues (I don't have that info handy...sorry).

That's all I can personally "enlighten" you with, other than to check out Peter Norton's "INside the IBM-PC and PS/2" (or similar volume).
