Server & Clients Specifications
 
Announcing the Beta Release of the ICQ Groupware


The ICQ Groupware, Beta Version (the "System") is composed of an ICQ Server Component, a database Component (see "Data Base Technical Requirements") and ICQ Client Component residing at user's desktop computer. The System can be deployed over any TCP/IP network including LAN, WAN, Internet and Dial-up networks.

COLLABORATIVE WORK

The System is designed to support communications and collaborative work over Intranet. It comes with many internal functions such as group chats, online and offline messaging, collaborative browsing, file transfer, central message broadcasting and more.

The System also supports many popular collaborating voice, video and data applications, which are launchable directly from user's Client Component of the ICQ Groupware, Beta Version. The Client Component of the ICQ Groupware, Beta Version has a built in mechanism for defining user's custom programs. Some of these applications are: VocalTec Internet Phone™, VDOLive VDOPhone™, Microsoft NetMeeting™, Netscape CoolTalk™, Intel lnternet Phone™, White Pine CU-SeeMe™, NetSpeak WebPhone™, M R A Associates, Inc. VidCall™, CineComM R A Associates, Inc. CineVideo AudioVision™, MM Art IrisPhone™, Connectix VideoPhone™, John Walker Speak Freely™ and more.

GROUPWORK

The System is designed to support groupwork. It supports multi-user chats, multi-user collaborating browsing and multi-user messaging. The user can construct a list of members of his group and see at all times their online/offline availability.

SERVER AND DATABASE FEATURES

Server and database Components' features are:

  1. Flexible Server User-Interface. System operator can define groups and departments in the organization. User can be included in a given group or department, to facilitate easy searching capabilities. User can communicate with every other user of any group in the organization.
  2. System Directory and Data Base Functions. Listings of user details, search by user details, messaging store-and-forward and more.
  3. Offline Messaging Store-and-Forward. Messages and URLs which sent to users while they are offline are saved in the data base and are forwarded automatically to them when they reconnect to the System.
  4. Online Services. These services are peer-to-peer and do not pass via the Server Component of the ICQ Groupware, Beta Version unless specially configured to do so (see "Client Features" and "Firewall Technical Requirements").
  5. Broadcasting. Messages can be broadcast to some or all of the users.
CLIENT FEATURES
  1. Personal Contact List. User can define a list of associates he wants to have on his contact list. The System will notify him automatically whenever his associates are online/offline, and if they are online, what their Status is.


  2. Status. The user has full control over the way his availability is presented to other users; He can select his availability Status (Online, Away, Do Not Disturb, Privacy (Invisible) and offline). All other users have a clear idea on his availability Status for collaboration, or other purposes, at all times.


  3. FileDirect. A file transfer module supports crash recovery, batch transfer, entire directory transfer and more.


  4. ChatDirect. Built in conference chat (both IRC and split-screen styles) supporting an unlimited amount of users.
  5. Online MessageDirect. The user can send his message to one or multiple users, forward them and reply.



  6. Offline MessageDirect Store and forward offline message support. Messages and URLs are saved in the Server Component of the ICQ Groupware, Beta Version while receiving user is offline, and delivered to him automatically when he logs in.
  7. Compose-Now Auto-Send Later. User can compose messages at any time and auto send them at a later time when he is connected to the net. He can select to send the messages only if the recipient user is online or whether he is on or offline.


  8. URLDirect - Collaborative Browsing. Invites recipients to join sender to go to a specific URL. The user can receive URLs which will launch his browser and auto-go directly to the incoming URL.


  9. Browser Compatibility. The Client Component of the ICQ Groupware, Beta Version supports Internet Explorer™ and Netscape Navigator™. User can launch browser from his Client Component of the ICQ Groupware, Beta Version and go directly to predefined URLs. The user can also add the received URLs to his bookmarks directly from his Client Component of the ICQ Groupware, Beta Version.
  10. History Record. All incoming and outgoing messages and URLs are automatically saved in user's Client for a future use. Incoming and outgoing messages are saved in separate folders. Dialogs can be retrieved in a threaded form. During, and at the end of the chat User can save chat to a text file, for a future use.
  11. Email Compatibility. supports most Internet and Intranet popular e-mail applications. The user can launch his email client and send messages directly from his Client Component of the ICQ Groupware, Beta Version. Client Component of the ICQ Groupware, Beta Version can check and alert automatically for most POP3 email servers, whether there are any and how many email messages are waiting for the user.

  12. User Interface. ICQ has a minimal footprint on user's desktop, and is designed to work concurrent with other applications. ICQ can also work in the background, off screen, on an auto incoming-message pop-up mode.
  13. Alerts, Accepts, Permissions And Privileges. Client Component of the ICQ Groupware, Beta Version is easily personalized and gives users a large variety of alert, accept, permission and privilege modes to select from.


  14. Bandwidth saver. Client Component of the ICQ Groupware, Beta Version uses minimum net resources.
  15. Personalized Greeting Message. Every user has his own Personalized Greeting Message, where he can leave messages and information to all other users.


  16. Preferences. Wide array of preferences enables the user to configure the Client Component of the ICQ Groupware, Beta Version to fit his individual needs.
SECURITY AND PRIVACY
  • Security, Permissions and Privileges. System operator can define access permissions on denials of individuals, or by range.
  • Online Peer-to-Peer Communication. Default online inter-users communication is peer to peer, providing faster response time and higher privacy. Server Component of the ICQ Groupware, Beta Version is employed only for offline messages (unless Server is intentionally selected to function on relay mode at online applications. See "Firewall Technical Requirements")
  • Password Access. Multiple users can log to the same machine. Access is password-controlled.
  • Ignore. The user can build his Ignore List which enables him to define which users' messages will be automatically ignored.
  • Authorization. The user can elect that his name will not be added to any other user's contact list unless his authorization is given.
  • Away. The user can choose to be in "Away" Status, globally or towards specific users. In this mode other users trying to send him a message will be alerted of his Status. The user can also define that whenever his computer's screen saver is on he will be seen as in "Away" mode.
  • Privacy (Invisible). The user can choose to be in a "Privacy (Invisible)" Status. In this mode he can see other users Status, but is being seen to other users as if he is offline. The user gets the messages sent to him as if he is in "Online" Status.
SERVER PLATFORM AND OPERATING SYSTEM
Mirabilis provides the Server Component of the ICQ Groupware, Beta Version Software. Platforms and Supporting Software are to be provided by Customer:
  1. Server Platform:
    • Intel x86-based systems (Pentium 166 or higher recommended).
    • 32 MB RAM (64-MB recommended).
    • 10 MB of disk space.
  2. Server Operating System: Windows NT 4.0.and above, on a server or a workstation.
COMMUINICATION TECHNICAL REQUIREMENTS
The Server Component of the ICQ Groupware, Beta Version should have a fixed IP defined by the system operator. System can be deployed over any TCP/IP based network, including LAN, WAN, Dial Up, ISDN, Ethernet etc.

CLIENT PLATFORM AND OPERATING SYSTEM

Mirabilis provides the Client Component of the ICQ Groupware, Beta Version Software. Platforms and Supporting Software are to be provided by Customer:

  1. Client Platforms: any 486DX machine, 8MB or higher, 3.5 MB of disk space.
  2. Client Operating System: Windows NT 4.0 or higher, Windows 95.
  3. Interoperability - different users may use various different operating systems on the same Intranet network.
DATABASE TECHNICAL REQUIREMENTS
  1. DataBase Software. The System can access any ODBC 2.0 compliant databases (Oracle, Sybase, Informix, SQL Server etc.). The System comes with Microsoft ODBC 3.0 Driver Manager subject to the terms and conditions of the ODBC license agreement that accompanies the System.
  2. DataBase Platform. Database software can reside at the same computer the ICQ Server resides at, or at any computer that is connected to it through Customer's net.
REGISTRATION AND CONTENT
  1. Data structure and Content. After the system operator defined the required ODBC connection (DSN) between the Server Component of the ICQ Groupware, Beta Version and the Customer database Component of the ICQ Groupware, Beta Version, the ICQ Data Base Wizard creates the necessary tables while the setup process is being made.
  2. Registration to the System. Each user and his details are registered and submitted individually, either by himself or by the system operator.
FIREWALL TECHNICAL REQUIREMENTS

The System can work from behind many firewalls, depending on the firewall configuration. No changes required in Customer's system configuration if Server and Clients Components of the ICQ Groupware, Beta Version are reside at the same side of the firewall. That is to say both following conditions are met:
1) No firewall exists between Clients and Server Components of the ICQ Groupware, Beta Version.
2) No firewall exists between any two Client Components of the ICQ Groupware, Beta Version.

If condition 1 is not met, firewall should be reconfigured by opening a port, or by making a port mapping. If condition 2 is not met, the desired port should be defined in both Client Component of the ICQ Groupware, Beta Version software and in the firewall, or a proxy SOCKS 4 or SOCKS 5 server can be used.

If a firewall exists between some two Client Components of the ICQ Groupware, Beta Version and the firewall can not be configured according to these requirements, only messages and URLs are available to the user in server relay mode, provided that all Clients are able to communicate with the server. In this mode all the traffic is relayed via the Server Component of the ICQ Groupware, Beta Version. Mirabilis technical department is available for further assistance.

The System will have no effect on the performance of the external collaborating voice, video, data, browsing and group chats applications, which user is launching by ICQ, while being connected through a firewall. Those applications will be subject to their own firewall configuration requirements and will perform irrespectively of the System.
