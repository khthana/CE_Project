2. Programs You'll Need

Naga

           Naga is an excellent tile editor by Ch33s3 and a few other guys. It supports many bitplanes, tile allignment, and many other fetures you may find necessary unfortunatly it only saves work to NAGA.SMC but you can just rename it. I highly reccomend this program.

X-Late

           This is BiGWeIRD's tanslation suite. It has graphical editing on the NES, Gameboy, TG16, and SNES. You can select up to 255 tiles at a time for whatever purpose you need them for. Once pasted as the bank appears, you can open it as a graphical text editor and go through looking for the words and everything else. This one now has pallette editing support and many other features to make it the #1 choice. Just remember, you need DOS4GW in the same dir and the command line is DOS4GW XLATE.

Hex Workshop

           I'm not gonna beat around the bush. Hex Workshop is the best Hex Editor anywhere! It has a great interface, and find/replace function (stay away from the all) function. It's also one of the easiest to use and best value around. If you're doing work with Hex, this is one godly tool. . Information on what it does and why you need it can be explained later.

Snestool

           Snestool is a utility used for many console operations (though the name suggests otherwise). It does anything and everything you need it to.. The only function covered with it in this guide is it's ability to make IPS patches. Don't let the name fool you, it makes IPS patches for anything including EXE files and NES ROMS. I think this tool is invaluble.

3. Bitplanes

           This is just a general guide to help you during tile editing. The topic, bitplanes. Bitplanes are just the layering of hex data. It overlaps the 1 bitplane data to make more colors possible in sprites, ect. Below is a list of where to find what and in what bitplane mode;

  • 16x16 tiles (dialogue, sometimes battle menus) = 1 bitplane (2 on occasion)

  • Kanji = Whatever your 16x16 was in

  • 8x8 Text = commonly 1 or 2 bitplane

  • Graphical Sprites = 4 bitplane for graphic intense, maybe 2 bitplane for signs, ect

Hope this clarify's the use of bitplane modes and helps you to understand where you need to look.

4. Changing Text

Using Hex Workshop

           This is real SNES ROM Hacking. Now, you first need to use a sprite editor and the Ranma RPG rom and our patch to find the text code in the ROM. To do so, just run X-Late, change to bitplane 2, and scroll through the endless ROM code until you find something like this (it's pretty far in);


this picture was taken right from X-Late. Each row is a number, 0 - F as is each column so number it all and then you can start working. Think of battleship, if it's in row 0 column A it would be 0A.

Now that you have your table, open up the ROM in Hex Workshop and click find. Enter the Hex bits for the thing you want to find. In order to find Okonmiaki you'll need to type "18 2E 32 31 30 2C 24 2E 2C" and hit find. Those will turn up highlighted so you're ready to enter your new bits. Go about doing this for the rest of the ROM as you proceed along with your translation.

Using X-Late

           Open up your rom and scroll through till you get to that same thing, this time, start on the 0 and drag a select all over the whole bank shown above hten paste it into the manu at the side. Change to text edit mode and begin scrolling through. It will use the data from your bank in place of letters so it's a graphical text editor. When you get to Okonmiaki, right click the letter you want to put in place of "O" and paste it over the "O".
Having fun yet?

16x16 Tiles

           16x16 Tiles work much like 8x8's but have a different system for layout. Below is an example of a 16x16 table from Final Fantasy 5;


Each of these tiles is assigned a hex bit just like the 8x8 tiles, the layout from this example was 3 tiles. One tile has the first part, the top half of the next tile has the rest of the first part, the bottom of that one has the first part of the 2nd half, the next tile has the rest of the 2nd half. X-Char alignes these when choosing kanji mode. This picture was taken from X-Char. Other methods of dialogue tiles are 4 8x8's each containing 1/4 of the tile. Another layout is 1 1/2 8x8's one being the top and one being the bottom of the tile. To avoid confusion I posted the pixel sizes and the layouts for each of the tiles below;

  • 16x16 - 4 8x8's each contain 1/4th the tile

  • 10x12 - 3 8x8s each containing part of the tile

  • 8x12 - 2 tiles, 1 has top half, 2nd has half of the bottom

  • 8x16 - 2 tiles each has half the 16x16

5. Changing Graphics

Using X-Late

           Editing graphics with X-Late is like using paintbrush. Here goes. Scroll through to the graphic as you did before. Now when you find it, right click on it then left click on the wokspace you want to drop it on (the big box to the left). Take a couple tiles, click the paint icon. It wil appear full screen with a 4 color palette from which to pick your colors. Pick your color and draw it onto the tile. When you finish that tile, hit the little door icon and it brings you back to the main screen, do as many as you please.

Using X-Late on Sprites

           Editing sprites with X-Late is probably the hardest thing in this guide. For this process you're going to need X-Late, to get various information about the graphic you are going to edit. For starters, load up Ranma RPG. Now scroll endlessly through the code until you find Genma (panda).

Now, click on the tile once to load it into the workspace near the palette and continue to load one whole body into it (each few tiles is part of a different position, make one). Now we're gonna turn Genma the panda into Genma the Polar Bear. Before you start to color it though, you must know that a pallete is stored somewhere in the rom which tells it what color equals what. If you just draw all over it, the image WILL come out killed. You need to find the pallete and see how it works. However, to keep it simple, we're making him all white. Now if you look in the game, white is white, pinkish is grey, and maroon is black. Knowing this, we can easily make him all white with a greyish shadow so it still looks good, now draw all over the body like I told you to, save it under another name and run it in an emulator. Genma now is a polar bear. Woo hoo!!! I;m sure if you've been here you'll notice I killed the gex section, as X-Late lets you modify palletes there is no more need for that. Don't ask for it back because it is all gone.

To deal with this problem, a simple yet innovative C program was written by J3d! with the purpose of going through and pulling out text from start to finish and spitting it out as a file. His original source (with a bit more code notation) can be downloaded here. After entering all your rom's info, you will need to compile the program. I would reccomend DJGPP by Delorie software as it is compact, effiecent, and most of all FREE! It is an excellent tool and since the time I downloaded it, I have not used anything else. Rom Expansion

           The complexity of this is far overrated. What I found to eb a very simple way to expand Ranma was simply this. Open your rom in a hex editor. You'll probably see a lot of "padding" of FF's of 00's at the end of your rom. This part is outside the code and is only there so it will check out okay witha n emultor. When you get a rom with a trainer or intro, 9 out of 10 times that intro is down here. Now having an extra 1k of text will not help you out very much so do this. First get UCON, go to dos prompt and type "UCON o [rom name]" and you should get a screen telling you all of the rom's information like the one I have shown below;

Now you need to open up your ROM in a hex editor and see where it ends at. From that point you'll start adding the 00's of FF's. The was it works is this;

 4.00 Mbit - $801F0 as last line of hex
 8.00 Mbit - $1001F0 as last line of hex
12.00 Mbit - $1801F0 as last line of hex
16.00 Mbit - $2001F0 as last line of hex
20.00 Mbit - $2801F0 as last line of hex
24.00 Mbit - $3001F0 as last line of hex
32.00 Mbit - $4001F0 as last line of hex
48.00 Mbit - $6001F0 as last line of hex

Now if you have a 12.00 Mbit rom and want it to be 16.00 Mbit, you would go to the last line of the rom's padding which would be $1801F0 and add 00's of FF's (depending on the rom) till you reach $2001F0. I have only done this on 3 roms but I'm pretty sure it would work on all roms. After this you need to work with the rom's pointer table to get text from the end. That I do not want to even get into in this document. This should at least help you move in the right direction.

7. Polishing Up

Creating IPS Patches

           If you're planning to give your work to someone or distribute your translation on the internet, you should make an IPS patch. Do this by opening up Snestool, clicking on Create IPS Patch, selecting the original, untranslated ROM, and then your version of it. That's all there is to it, you now have an IPS patch to distribute freely!
