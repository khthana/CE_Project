บทคัดย่อ

ขอบเขตของรายงานนี้จะเป็นเรื่องของการทำความรู้จักกับคอร์บา(CORBA) ในส่วนของสถาปัตยกรรม การทำงาน และส่วนประกอบต่างๆ ที่สำคัญ อย่างในเรื่องของการทำงานของ ORB (Object Request Broker) ที่มีการทำงานเมื่อเกิดการร้องขอจากผู้ขอใช้บริการ ซึ่งมีได้ทั้งแบบแน่นอนและแบบเปลี่ยนแปลงได้ SII\DII(Static and Dynamic Invocation) หรือจะเป็นแบบการใช้บริการของ IR(Interface Repository) การทำงานร่วมกันระหว่าง ORB Interoperability ภาษาที่ใช้ในการติดต่อระหว่าง ORB ที่เรียกว่า GIOP(General Inter-ORB Protocol) และการแปลงจาก ภาษาที่ใช้ระหว่าง ORB เป็นข้อมูลที่ใช้ติดต่อกันระหว่างเครื่องเป็น TCP/IP โดยผ่านระบบเครือข่ายที่เรียกว่า IIOP(Internet Inter-ORB Protocol) รวงทั้งการศึกษาและทำการทดลองโดยจะมีตัวอย่างของโปรแกรมที่มีการทำงานเฉพาะภายในเครื่องเดียวกัน

Abstract

Scope of this paper is understanding the CORBA, and important to architecture, operate and component. Such as operate of the ORB (Object Request Broker) to operate when have invocation from Client. So invocation have specification in Static Invocation Interface and Dynamin Invocation Interface or Interface Repository service. Interoperability of between ORB. Protocol of interface inter ORB is GIOP (General Inter-ORB Protocol) and mapping protocol GIOP to TCP/IP across thorough network is call IIOP (Internet Inter-ORB Protocol). Inculde study and to try in example program to testing in singel machine.
