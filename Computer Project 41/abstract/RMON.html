บทคัดย่อ

บทความนี้จัดทำขึ้นเพื่ออธิบายถึงโครงสร้างและการทำงานของมาตรฐานที่ใช้ในการบริหารและควบคุมระบบเครือข่ายคือ RMON (Remote Network Monitoring) ซึ่งเป็นมาตรฐานใหม่ที่มีลักษณะคล้าย ๆ กับ SNMP โดยใช้ฐานข้อมูล MIB เหมือนกัน แต่สามารถใช้งานได้อย่างมีประสิทธิ เนื้อหาในบทความนี้จะกล่าวถึงหลักการโดยทั่ว ๆ ไปของตัว RMON ที่มีลักษณะที่ดีกว่า SNMP มีการแบ่งฐานข้อมูลออกเป็นกลุ่ม ๆ ทั้งหมด 9 กลุ่มในระบบ Ethernet และ 10 กลุ่มสำหรับระบบ token Ring และ ในแต่ละกลุ่มนั่นยังมี Object ต่าง ๆ ที่ใช้ในการแสดงถึงสถานะของการทำงานขณะนั้นของ RMON ซึ่งในเครือข่ายนั้นจะมีตัว RMON Client ได้หลายเครื่องก็ได้แต่ไม่จำเป็นต้องมีคุณสมบัติต่าง ๆ ที่สูงเหมือนกับเครื่องคอมพิวเตอร์ที่ทำงาน ด้วยฟังก์ชั่น SNMP Server โดยแต่ละเครื่องสามารถที่จะกำหนดให้แสดงผลในแต่ละกลุ่ม ก็ได้ ทำให้การควบคุมระบบเครือข่ายขนาดใหญ่ได้อย่างมีประสิทธิภาพ

Abstract

This report explains about structure and working standard for managing and monitoring the network system, which is called RMON. It is a modern standard that uses MIB as its database like SNMP, but it works more efficiently. This report also explains about the difference between RMON and SNMP how it is superior than the latter. In Ethernet system, database is divided into 9 groups. In Token ring system, it is also divide into 10 groups. Each groups have object whose capacity is to show the working condition of RMON. Several RMON Client are in the network but they are unnecessary to have the qualification like the computer that use SNMP sever function. Each computer, which has RMON Client, can determine each group to perform the result. It is more efficient to control large network system.
