Interprocess communication

บทคัดย่อ

ในระบบปฏิบัติการ Unix แบ่งการทำงานออกเป็นส่วนย่อย ๆ แล้วจัดสรรงานในแต่ละส่วนให้ Process ไปทำงาน เพื่อให้งานในแต่ละส่วนสามารถนำประกอบกันจนได้เป็นงานที่เสร็จสมบูรณ์จึงจำเป็นต้องมีการศึกษา การติดต่อสื่อสารแลกเปลี่ยนข้อมูลกันระหว่าง Process เพื่อให้มีการใช้ข้อมูลร่วมกัน ซึ่งวิธีการในการติดต่อ สื่อสารเพื่อแลกเปลี่ยนข้อมูลที่นิยมใช้ในระบบปฏิบัติการ Unix เรียกกันว่า IPC ( Interprocess Communication ) บทความนี้จะนำเสนอกรรมวิธีในการติดต่อสื่อสารเพื่อแลกเปลี่ยนข้อมูล ( IPC ) ซึ่งมีรูปแบบพี้นฐาน ของ IPC ที่นิยมใชัอยู่ด้วยกัน 5 รูปแบบ ได้แก่ pipes, FIFOs, Message queues, Semaphores และ Shared memory เพื่อความเข้าใจในกรรมวิธีต่าง ๆ ของ IPC ผู้อ่านควรมีความรู้ความเข้าใจในระบบปฏิบัติการ Unix เบื้องต้น ซึ่งเป็นเนื้อหาส่วนแรก ส่วนเนื้อหาส่วนที่สองจะเป็นเนื้อหาเกี่ยวกับ IPC

Abstract

Unix system separate task to several small modules call "Process". Each process will exchange data each other. Therefor, Learning about communication between the Processes is essential, in another name " Interprocess Communication (IPC) ". This article will represent about method of IPC, which has 5 method following : pipes, FIFOs, Message queues, Semaphores and Shared memory. Before learning about IPC, it is important to know about basic definition and some system call of Unix in section 1. later learn about IPC in section 2.
