บทคัดย่อ

ในการส่งข้อมูลระหว่างกันอย่างปลอดภัยจะต้องมีการเข้ารหัส การเข้า-ถอดรหัสแบบสามัญจะมีการใช้กุญแจดอกเดียวกันทั้งในการเข้าและถอดรหัส ทำให้บุคคลที่สามซึ่งรู้กุญแจดังกล่าวสามารถถอดรหัสข้อความนั้นได้ ทำให้ไม่มีความปลอดภัยในการส่งข้อมูลเท่าที่ควร การเข้า-ถอดรหัสแบบกุญแจสาธารณะจะมีการใช้กุญแจสองดอก คือกุญแจสาธารณะและกุญแจส่วนบุคคล ในการเข้ารหัสจะใช้กุญแจสาธารณะซึ่งมีการแพร่กระจายไปทั่ว และในการถอดรหัสจะใช้กุญแจส่วนบุคคลที่จะต้องเก็บไว้เป็นส่วนตัว ทำให้บุคคลที่เราต้องการส่งถึงเท่านั้นถอดรหัสได้ ในการแพร่กระจายกุญแจสาธารณะจะมีหลายวิธีคือ การแพร่กระจายกุญแจสาธารณะด้วยตนเอง, ไดเร็คทอรีที่เข้าถึงได้โดยสาธารณะ, แหล่งของกุญแจสาธารณะที่เชื่อถือได้ และ การรับรองกุญแจสาธารณะ ซึ่งวิธีการรับรองกุญแจสาธารณะจะเป็นวิธีที่มีประสิทธิภาพดีที่สุด การรับรองกุญแจสาธารณะทำโดยหน่วยงานรับรอง ซึ่งรูปแบบของการรับรองจะกำหนดโดยมาตรฐาน X.509 การทำการรับรองจะมีการนำมาใช้ในหลายๆโปรแกรมเช่นในการส่งจดหมายอิเล็คทรอนิคส์ คือ PEM และมีการนำมาใช้ในระบบความปลอดภัยใน Browser คือ SSL อีกด้วย

Abstract

In secure transaction, data must have been encrypted. In conventional cryptography, only one key is used for both encryption and decryption. So the third-party who know this key will be able to decrypt this message. The public-key cryptography uses of two difference keys a public- key and private-key. We use a public-key, which is distributed publicly, to encrypt a message. And we use a private-key, which is kept privately, for decrypt the message. There are several methods for public-key distribution : Public Announcement of Public-Key, Public Available Directory, Public-key authority and Public-Key Certificate. The public-key is certificated by certificate authority . The format of certificate is set by X.509 standard. Public-key certification concept is used to implement in several programs such as PEM and SSL.
