Secure Socket Layer

นาย วรุตม์ สุดภู่ทอง

บทคัดย่อ
	เนื่องจากการเจริญเติบโตของอินเทอร์เน็ตและการสื่อสารข้อมูลทางดิจิตอล โปรแกรมประยุกต์ต่าง ๆ 
จำนวนมากต้องการส่งข้อมูลอย่างปลอดภัยไปยังคอมพิวเตอร์ระยะไกล เอสเอสแอล (SSL) ถูกออกแบบมาเพื่อใช้
ในการแก้ปัญหานี้ในมาตรฐานเปิด
	ซีเคียวซ็อกเก็ตเลเยอร์ (Secure Socket Layer) ถูกพัฒนาโดยเน็ตสเคปคอมมิวนิเคชันคอร์ปอเรชัน 
(Netscape Communications Corporation) เอสเอสแอลเป็นโพรโตคอลที่ใช้สำหรับการสื่อสารระหว่างจุดต่อจุด 
เอสเอสแอลจัดให้มี การเข้ารหัสข้อมูล, การพิสูจน์ตนของเซิร์ฟเวอร์, การตรวจสอบความสมบูรณ์ของข้อมูล และ
การพิสูจน์ตนของไคลเอนต์ สำหรับการติดต่อโดยทีซีพี/ไอพี (TCP/IP) เนื่องจากเอสเอสแอลถูกสร้างรวมเข้าไว้
ในบราวเซอร์และเว็บเซิร์ฟเวอร์หลัก ๆ ข้อดีของเอสเอสแอล คือ ติดตั้งใบรับรองดิจิตอลได้ง่าย
	เอสเอสแอลแบ่งตามความแข็งแรงของคีย์ (Key) ที่ใช้ในการเข้ารหัสเป็น 2 ชนิด คือ คีย์ที่ใช้ในการเข้า
รหัสขนาด 40 บิตและ128 บิต ยิ่งคีย์ที่ใช้ในการเข้ารหัสมีความยาวมากขึ้นเท่าไรความยากที่จะถอดรหัสก็เพิ่มขึ้น
มากเท่านั้น บราวเซอร์ส่วนมากสนับสนุนคีย์ที่ใช้ในการเข้ารหัสขนาด 40 บิตและบราวเซอร์รุ่นใหม่รวมทั้งเน็ตส
เคปเนวิเกเตอร์เวอร์ชัน 4.0 อนุญาติให้ผู้ใช้สามารถเข้ารหัสโดยใช้คีย์ขนาด 128 บิตซึ่งมีความแข็งแรงกว่าแบบ 40 
บิต เวอร์ชันล่าสุดของเอสเอสแอลถูกเรียกว่า ทรานสปอร์ตเลเยอร์ซีเคียวริตี้

Abstract
With the growth of the Internet and digital transmission, many applications need to securely transmit 
data to remote applications and computers. SSL was designed to solve this problem in an open standard.
	Secure sockets Layer was developed by Netscape Communication Corporation. SSL is the Internet 
protocol for point-to-point connections. It provides data encryption, server authentication, message integrity and 
optional client authentication for a TCP/IP connection.  Because SSL is built into all major browsers and web 
servers, simply installing a digital certificate turns on their SSL capabilities.
	SSL comes in two strengths, 40-bit and 128-bit, which refer to the length of the session key  
generated by every encrypted transaction. The longer the key, the more difficult it is to break the encryption 
code. Most browsers support 40-bit SSL sessions, and the latest browsers, including Netscape Communicator 
4.0, enable users to encrypt transaction in 128-bit sessions  trillions of times stronger than 40-bit sessions.
The latest enhancement of SSL is called Transport Layer Security, TLS.
